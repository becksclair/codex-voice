## Why this matters

The workspace compiles **two versions of cpal** — 0.15.3 (pinned directly, used by the recorder) and 0.17.1 (forced by rodio 0.22 for playback) — dragging duplicate ALSA bindings (`alsa 0.9.1` + `0.10.0`). Beyond build time and binary size, recording and playback run on divergent audio-layer versions with different device-enumeration behavior. Two smaller items ride along: `codex-voice-tts` bypasses the workspace serde pin (the only inline dep version in the workspace), and rodio's default features compile the entire symphonia decoder suite (flac/mp3/aac/vorbis/isomp4/ogg) to support exactly two lines of playback code.
